# Agentalk Hardening Roadmap

Date: 2026-05-02

This file records follow-up changes recommended after fixing the multi-line pipe runaway that caused excessive Claude Code usage.

## Executive Summary

The immediate runaway path has been blocked: multi-line piped input no longer drives the interactive REPL line by line, and headless modes now have explicit `--stdin` / `--file` entry points.

The next work should make Agentalk safer by default. The most important theme is to make expensive model fan-out explicit, observable, and budgeted.

## P0: Add Per-Run Call Budgets

Recommended files:

- `lib/agents.js`
- `lib/discuss.js`
- `index.js`

Problem:

Even with stdin fixed, a malformed moderator plan or broad all-agent default can still create more calls than expected. Claude is especially sensitive because it may be the user's main paid quota.

Recommendation:

- Add a run budget object shared across one Agentalk invocation.
- Track total agent calls and per-agent calls.
- Default limits:
  - `maxTotalAgentCalls`: 12
  - `maxClaudeCalls`: 3
  - `maxModeratorCalls`: 4
- Add CLI overrides:
  - `--max-calls N`
  - `--max-claude-calls N`
  - `--no-budget` only for explicit local debugging
- When the budget is exceeded, stop launching new agents, save a summary, and print a clear warning.

Why this matters:

This caps blast radius even if another route accidentally fans out.

## P0: Exclude Claude By Default When Caller Is Claude Code

Recommended files:

- `lib/config.js`
- `lib/discuss.js`
- `index.js`
- `mcp-server.js`

Problem:

When Claude Code invokes Agentalk and Claude is also an active participant or moderator, the main model can recursively spend its own quota. This undermines the goal of spreading load across subscriptions.

Recommendation:

- Detect likely Claude Code caller context:
  - `CLAUDECODE` / Claude-related environment signals if available
  - `--from-claude`
  - MCP caller identity where known
- In that context, exclude `claude` from default participant lists.
- Require explicit opt-in:
  - `--include-claude`
  - `/agents include-claude-on-caller claude`
- If the configured moderator is `claude`, choose the next active non-Claude agent or require explicit confirmation.

Why this matters:

The user's main model should not be silently consumed by a tool intended to offload work.

## P1: Make MCP Defaults Safer

Recommended file:

- `mcp-server.js`

Problem:

The MCP `ask`, `discuss`, and `debate` tools currently default to all active agents when `agents` is omitted. In an MCP context, an agent may omit `agents` casually and trigger expensive fan-out.

Recommendation:

- Change `ask` so `agents` is required, or default to non-main preferred subscription agents only.
- Update tool descriptions to say:
  - "Specify agents explicitly."
  - "Omitting agents is refused or uses safe defaults."
- For `discuss` / `debate`, consider defaulting to `codex,gemini,opencode` and excluding Claude unless requested.

Why this matters:

MCP calls are often generated by another agent, so the API should be conservative.

## P1: Add Run Audit Logs

Recommended files:

- `lib/agents.js`
- new `lib/run-audit.js`

Problem:

Today, diagnosing usage requires reading Claude Code JSONL logs and Agentalk summaries. That is too indirect.

Recommendation:

Create append-only JSONL logs:

```text
~/.agentalk/runs/YYYYMMDD.jsonl
```

Each Agentalk invocation should record:

- run id
- cwd
- mode
- topic preview
- caller hints
- selected participants
- moderator
- every agent spawn with timestamp, elapsed, token estimate, status
- budget stop reason if any
- summary path

Why this matters:

The user should be able to answer "why did this consume quota?" without forensic work.

## P1: Add A Direct Refusal For REPL Batch Commands

Recommended files:

- `lib/input.js`
- `index.js`

Problem:

The current fix refuses multi-line piped input. A one-line pipe is still accepted for compatibility. That is acceptable short-term, but there is still ambiguity around using pipes to automate REPL behavior.

Recommendation:

- Add a startup warning for any non-TTY REPL use.
- Consider refusing all piped input unless one of these is true:
  - a headless mode is present
  - `--allow-piped-repl` is passed
- Keep `--allow-piped-repl` undocumented or clearly marked as unsafe.

Why this matters:

Interactive REPL semantics should not be driven by scripts unless the user opts into that risk.

## P1: Tighten Product Boundary Between `agentalk` And `agentalk-delegate`

Recommended files:

- `README.md`
- `skills/agentalk.md`
- `skills/agentalk-delegate.md`
- `scripts/install-skill.js`

Problem:

The previous incident happened because an implementation sub-task was sent through the discussion tool. Docs now warn against it, but the boundary should be more explicit across install surfaces.

Recommendation:

- In `/agentalk` skill, add examples of what not to do:
  - "fix this test"
  - "edit these files"
  - "read these 20 files and summarize"
- In `/agentalk-delegate` skill, add "migration from bad pattern":
  - Bad: `cat task.txt | agentalk`
  - Good: `agentalk-delegate gemini "..." --files "..."`
- Re-run `agentalk-delegate init` or `node scripts/install-skill.js` after release so installed Claude Code skills receive the updated rules.

Why this matters:

Agents learn from nearby tool instructions. The correct path must be obvious.

## P2: Add Headless Smoke Tests

Recommended files:

- new `test/cli-smoke.sh`
- `package.json`

Problem:

There is no test suite. This allowed a dangerous CLI behavior to persist unnoticed.

Recommendation:

Add a lightweight shell smoke test that does not call real agents:

- `node --check index.js`
- `node --check lib/input.js`
- `printf 'a\nb\n' | node index.js` exits 2 and contains "Refusing"
- `node index.js --deepen` exits 2 and contains "Missing topic"
- `node index.js --deepen --file` exits 2 and contains "--file requires a path"

For headless behavior that would normally spawn agents, add a mock-agent config fixture later.

Why this matters:

The highest-risk behavior is CLI parsing, and it can be tested without spending model quota.

## P2: Improve Headless Argument Parsing

Recommended file:

- `index.js`

Problem:

The current parser is still hand-rolled. It now supports `--stdin`, `--file`, and `--agents`, but more flags will increase fragility.

Recommendation:

- Add a small internal parser function with explicit token consumption.
- Validate unknown flags in headless mode.
- Add support for:
  - `--rounds N`
  - `--turns N`
  - `--max-calls N`
  - `--include-claude`
- Consider a CLI parser dependency only if the project accepts the extra dependency.

Why this matters:

Clear parsing prevents surprising invocation behavior by coding agents.

## P2: Consider `agentalk --ask` Only If Product Boundary Needs It

Recommended files:

- `index.js`
- `README.md`

Problem:

There is a temptation to add `agentalk --ask gemini --stdin` as a convenience. That would solve direct single-agent asks in `agentalk`, but it may blur the line with `agentalk-delegate`.

Recommendation:

Do not add it unless there is a clear product reason. Prefer `agentalk-delegate` for direct work. If added, make it read-only/advisory by default and document that implementation sub-tasks still belong to `agentalk-delegate`.

Why this matters:

The previous incident was partly caused by product boundary confusion.

## Suggested Next Commit Scope

If doing one more safety pass, implement these together:

1. Per-run call budgets.
2. Claude exclusion when caller is Claude Code.
3. MCP safe defaults.
4. CLI smoke tests.

That would turn the current fix from "the known pipe incident is blocked" into "future runaway paths have bounded blast radius."
